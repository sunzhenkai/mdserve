# 文档页 UI 优化实施计划

## 目标

按你的口径完成 3 项改造：

- 顶部导航改为“悬浮风格”（非 fixed）
- 导航下方重构为三栏卡片布局（左文档树 / 中间信息+正文双卡 / 右大纲）
- 标签/分类弹窗在视觉与布局上对齐搜索弹窗，并抽取可复用弹层骨架组件

## 改造范围与方式

### 1) 悬浮导航（仅视觉悬浮，不用 fixed）

在 `[/home/wii/workspace/public/mdserve/web/src/App.tsx](/home/wii/workspace/public/mdserve/web/src/App.tsx)` 调整顶部 header 容器：

- 保持文档流布局，不使用 `fixed`
- 在页面内容区外层加入顶部留白与左右留白
- 给 header 增加圆角、阴影、半透明背景与轻微模糊（与当前主题变量兼容）
- 确保移动端按钮（菜单/搜索/大纲）与桌面导航菜单在新 header 中对齐正常

### 2) 导航下方三栏 Card 布局

在 `[/home/wii/workspace/public/mdserve/web/src/App.tsx](/home/wii/workspace/public/mdserve/web/src/App.tsx)` 重构 `main` 区域结构：

- 将左、中、右三块都包在 card 容器中（统一圆角+阴影+边框）
- 左侧 card：承载 `FileTree`（保留现有折叠逻辑）
- 中间区域拆为上下两个 card：
  - 上卡：`DocumentInfo`
  - 下卡：`MarkdownViewer`（可滚动主体）
- 右侧 card：承载 `Outline`（右侧明确为大纲）
- 保留移动端 `Sheet` 行为，仅调整桌面端布局样式，避免功能回归

### 3) 标签/分类弹窗视觉与搜索弹窗统一

新增一个通用弹层壳组件（最小抽象，先统一视觉骨架）：

- 新文件：`[/home/wii/workspace/public/mdserve/web/src/components/common/ModalShell.tsx](/home/wii/workspace/public/mdserve/web/src/components/common/ModalShell.tsx)`
- 职责：统一 `DialogContent` 的宽高、圆角、阴影、遮罩、头部区域、主体滚动区、可选底部区域

然后改造：

- `[/home/wii/workspace/public/mdserve/web/src/components/SearchModal.tsx](/home/wii/workspace/public/mdserve/web/src/components/SearchModal.tsx)`
  - 用 `ModalShell` 承载现有内容，保持现有搜索交互不变
- `[/home/wii/workspace/public/mdserve/web/src/components/TagsModal.tsx](/home/wii/workspace/public/mdserve/web/src/components/TagsModal.tsx)`
  - 用 `ModalShell` 改造外观结构，使其与搜索弹窗在视觉层级一致
  - 保留现有 Tabs 与左右内容面板逻辑（本次不强行统一行为）

## 验证与回归检查

- 桌面端：
  - 悬浮 header 在滚动正文时保持视觉悬浮效果
  - 三栏卡片阴影/圆角一致，左右栏和中间双卡布局稳定
  - 右侧大纲点击跳转功能正常
- 移动端：
  - 左菜单/右大纲 Sheet 与搜索弹窗打开关闭正常
- 弹窗一致性：
  - 搜索与标签/分类弹窗的边框、圆角、遮罩、头部区风格统一
  - 功能行为无退化（搜索、防抖、标签/分类选择与跳转）

## 风险与控制

- `main` 区域重排可能影响滚动层级：优先保持“正文区单独滚动”语义
- card 化后空间密度变化较大：通过响应式断点和内边距微调保证可读性
- 通用组件抽象过深会增加复杂度：本次只抽“壳层”避免过度设计
